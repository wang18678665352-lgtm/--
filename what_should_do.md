# 电子医疗管理系统 - 开发规则约束（ClaudeCode 专用）
## 1. 项目基本信息
- 项目名称：电子医疗管理系统
- 适用场景：医院信息化管理，替代纸质登记与人工统计
- 开发角色：患者、医生、管理员三端分离开发
- 核心设计思想：三角色登录、分权限操作、全过程记录
必须要用链表实现
必须要用c实现
不能使用数组，除非是字符串数组
多端共享数据，不能出现不同端的可观看共享信息有差异
---

## 2. 角色与权限约束（强制）
### 2.1 角色定义
系统仅允许三类用户：**患者、医生、管理员**
- 患者：仅操作本人信息
- 医生：仅操作本人接诊/负责患者
- 管理员：全局数据维护权限，关键操作需二次确认

### 2.2 权限边界（禁止越权）
- 患者禁止访问医生/管理员界面
- 医生禁止修改科室、医生、药品、病房基础数据
- 管理员禁止模拟诊疗操作
- 所有跨角色访问直接拒绝并提示权限不足

---

## 3. 功能开发约束
### 3.1 患者端（必须实现）
1. 登录验证（账号+密码+角色）
2. 按科室/医生/日期挂号，生成挂号记录
3. 查询挂号状态：待就诊/已就诊
4. 查询本人诊断结果、处方、住院信息
5. 查看个人治疗进度
6. 仅展示本人数据，禁止查看他人信息

### 3.2 医生端（必须实现）
1. 登录后自动显示本人待接诊挂号
2. 新挂号实时提醒
3. 接诊：录入诊断、治疗建议、检查项目
4. 开具处方，自动扣减药品库存
5. 校验重复用药、库存不足提示
6. 病房呼叫提醒接收
7. 标记紧急病人（优先分配床位/药品）
8. 更新患者治疗进度，防止重复医疗

### 3.3 管理员端（必须实现）
1. 科室管理：增删改查科室信息
2. 医生管理：维护信息、工作量统计
3. 患者管理：档案、住院状态、治疗阶段
4. 药品管理：价格、库存、预警线、报销规则
5. 病房管理：总床位、剩余床位、预警线
6. 报表统计：挂号量、就诊量、出入库、使用率、报销
7. 药品/床位低于阈值自动预警
8. 数据补货与资源调配

### 3.4 公共扩展功能（必须实现）
1. 非法输入拦截：空值、负数、重复编号、错误日期
2. 医疗记录统一管理：门诊/处方/住院/治疗/出院
3. 药品库存预警 + 病房床位预警
4. 医生繁忙度推荐（挂号时推荐空闲医生）
5. 用药智能推荐
6. 差异化报销：按患者类型×药品设置报销比例


---

## 5. 算法与逻辑约束
1. **医生推荐算法**：按挂号数+在治患者+未完成任务计算繁忙值，推荐低繁忙医生
2. **病房预警**：剩余床位 ≤ 预警线 → 管理员预警；紧急病人优先
3. **药品预警**：库存 ≤ 预警线 → 补货提醒；开药时校验库存
4. **防重复医疗**：同治疗阶段/短期重复用药 → 提示并二次确认
5. **报销计算**：报销金额=总价×比例；自费=总价−报销金额
6. **数据唯一性**：ID 全局唯一，禁止重复
7. **数值约束**：数量、金额必须 ≥0

## 6. 流程约束
### 6.1 系统总流程
登录 → 身份校验 → 角色判断 → 进入对应子系统 → 业务处理 → 输入校验 → 数据更新 → 返回结果

### 6.2 患者流程
登录 → 选科室/医生 → 挂号 → 医生端提醒 → 就诊后查询结果

### 6.3 医生流程
登录 → 查看挂号提醒 → 接诊 → 录入诊断 → 开处方 → 更新进度 → 处理病房呼叫

### 6.4 管理员流程
登录 → 数据维护 → 处理预警 → 生成报表

---

## 7. 输入输出约束
### 7.1 输入约束
- 禁止空值（必填项）
- 禁止负数（库存、金额、床位）
- 日期必须符合标准格式
- ID 唯一不重复
- 非法字符自动拦截

### 7.2 输出约束
- 操作成功/失败明确提示
- 失败需返回原因
- 界面简洁，权限内数据可见
- 预警信息醒目推送

---

## 8. 测试验收约束
必须通过以下测试：
1. 三角色登录正确性，错误密码/角色拒绝
2. 患者挂号 → 医生端实时提醒
3. 患者仅查看本人信息
4. 开药自动扣库存，库存不足提示
5. 管理员可维护所有基础数据
6. 权限越权访问拒绝
7. 药品/床位低于阈值正确预警
8. 不同患者类型报销比例正确

---

## 9. 代码规范
1. 模块化开发：登录、患者、医生、管理员、公共服务
2. 三层架构：用户层→业务逻辑层→数据存储层
3. 权限判断统一封装，禁止硬编码权限
4. 所有数据库操作加异常捕获
5. 预警、提醒、校验逻辑独立封装

---

## 10. 禁止行为
- 禁止不做权限判断直接展示数据
- 禁止药品/床位不做库存校验直接扣减
- 禁止跨角色数据访问
- 禁止无输入校验直接入库
- 禁止忽略预警逻辑
- 禁止治疗进度不更新导致重复医疗
